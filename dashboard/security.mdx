## Antinuke

Antinuke watches for dangerous actions like channel deletion, role deletion, or mass bans.

- Enable or disable Antinuke from the configuration section.
- Set a log channel for Antinuke alerts.
- Add Antinuke admin roles or users that should bypass restrictions.
- Configure protected actions from the **Roles**, **Server**, and **Special** tabs.
- Add up to three rules per action, each with a threshold, time window, punishment, and optional undo behavior.
- Whitelist specific roles or users for an action when their activity should not count toward a rule.

The **Roles** tab covers role creation, deletion, editing, and role assignment changes. The **Server** tab covers channel creation or deletion, bans, kicks, and emoji creation or deletion. The **Special** tab includes Vanity URL changes and Bot Protection, which can automatically kick newly joined bots unless they are on a whitelisted bot ID list.

## Antiraid

Antiraid is designed for mass-join events and other suspicious bursts of activity.

- Enable or disable Antiraid and choose a log channel.
- Configure **Mass Join Detection** with a join threshold, time window, punishment, and target mode.
- Choose whether mass-join punishments apply only to members over the limit or to all accounts involved in the raid window.
- Lock the server during a raid by assigning a configured role.
- Enable auto-punishment during an active raid.
- Punish accounts with no avatar when that is a useful risk signal for your server.
- Set a minimum account age and punishment for accounts younger than that limit.

## AI Moderation

AI Moderation adds an extra review layer for harmful content so your team can catch issues faster without manually reading every message.

### Configuration

- Turn scanning on or off for the server.
- Pick a log channel for flags, deletions, and review actions.
- Set the maximum number of checks per minute to control how much content is processed.
- Set exceptions for channels, roles, or users that should not be scanned.
- Use exceptions for staff-only channels, trusted automation, or areas where message scanning is not appropriate.

<Image src="/assets/images/dashboard/security-ai-moderation-1.png" alt="AI-Moderation Page" size="sm"/>

### Categories and responses

Each category can be tuned separately, so you do not have to use one global sensitivity level.

- **Hate speech** can flag slurs, dehumanizing language, and abusive insults.
- **Harassment** helps catch bullying and targeted abuse.
- **Self-harm** can surface messages that may need urgent moderator attention.
- **Sexual content** helps reduce explicit or unsafe content in member spaces.
- **Violence** can detect threats or graphic violent language.

For each category, you can adjust the sensitivity slider, choose whether flagged messages should be deleted, and define escalation rules for repeat offenses.

### Escalation rules

Escalation rules convert repeated AI moderation detections into staff actions.

- Set an infraction threshold for the rule.
- Choose the time window used to count those infractions.
- Pick a punishment such as timeout, kick, or ban.
- Add a duration when the punishment supports one.
- Use stricter rules only for categories where false positives are rare in your community.

## Recommended rollout

<Steps>
  <Step title="Start with logs">
    Turn on logging first so you can see what would trigger before using harsher punishments or automatic deletions.
  </Step>
  <Step title="Whitelist trusted staff">
    Add owners, lead admins, approved automation roles, and channels that should be excluded before enforcing protections.
  </Step>
  <Step title="Tighten thresholds">
    After a few days of observing normal activity, adjust thresholds and AI sensitivity until they match your moderation pace.
  </Step>
</Steps>

<Callout type="warning">
  Aggressive thresholds or AI settings can punish legitimate activity. Start conservative, review detections regularly, then tighten settings after you understand your server's normal patterns.
</Callout>
