## Callout

Colored hint boxes with icon. Available types: `info`, `tip`, `warning`, `error`, `success`.

<Callout type="info">
  This is an informational callout. Use it to highlight important details.
</Callout>

<Callout type="tip">
  You can use callouts to share tips and best practices with your readers.
</Callout>

<Callout type="warning">
  Be careful — this action cannot be undone.
</Callout>

<Callout type="error">
  Something went wrong. Please check your configuration.
</Callout>

<Callout type="success">
  Your changes have been saved successfully.
</Callout>

## Tabs

Tabbed content for showing code in multiple languages or different approaches.

<Tabs items={["JavaScript", "TypeScript", "Python"]}>
  <Tab>
    ```js
    const greeting = "Hello, World!";
    console.log(greeting);
    ```
  </Tab>
  <Tab>
    ```ts
    const greeting: string = "Hello, World!";
    console.log(greeting);
    ```
  </Tab>
  <Tab>
    ```python
    greeting = "Hello, World!"
    print(greeting)
    ```
  </Tab>
</Tabs>

## Accordion

Collapsible sections for FAQs or optional details.

<Accordion>
  <AccordionItem title="How do I change the command prefix?">
    Go to your server's Dashboard, navigate to Settings, and update the prefix field. The default prefix is !.
  </AccordionItem>
  <AccordionItem title="Can I use Bender on multiple servers?">
    Yes! Bender can be added to as many servers as you want. Each server has its own independent configuration.
  </AccordionItem>
  <AccordionItem title="Is Bender free to use?">
    Bender is free for all core features. Premium features like custom branding and priority support are available with a subscription.
  </AccordionItem>
</Accordion>

## Badge

Inline status labels. Available variants: `default`, `info`, `success`, `warning`, `error`.

- This command is <Badge variant="success">New</Badge> in the latest update.
- Requires <Badge variant="warning">Admin</Badge> permissions to use.
- This feature is <Badge variant="error">Deprecated</Badge> and will be removed.
- Available in <Badge variant="info">Beta</Badge> for testing.
- Status: <Badge>Stable</Badge>

## Image

Images are always centered and fill the available width. Use `size` to control the max-width.

**Props:** `size` (`sm` | `md` | `lg` | `full` or CSS value), `caption`, `description` (adds a frame), `rounded`, `noRadius`.

<Image src="/images/icon.png" alt="Bender Logo" size="sm" caption="Small" />

<Image src="/images/icon.png" alt="Bender Logo" size="sm" description="The Bender bot logo — used across all platforms and branding materials." />

## Image Carousel

Multiple images with left/right navigation and dot indicators.

**Props:** `size` (`sm` | `md` | `lg` | `full` or CSS value), `noPadding` (edge-to-edge), `noRadius`.

<ImageCarousel size="sm">
  <CarouselSlide src="/images/icon.png" alt="Slide 1" />
  <CarouselSlide src="/images/icon.png" alt="Slide 2" />
  <CarouselSlide src="/images/icon.png" alt="Slide 3" />

## Advanced Markdown

Examples of standard Markdown features that are styled by the prose system.

### Inline formatting

This is **bold text**, *italic text*, and `inline code`. You can also use [links](https://bender.best) and ~~strikethrough~~.

### Blockquote

> This is a blockquote. It's useful for highlighting quotes or important notes that don't need the full callout treatment.

### Lists

- Works great for feature lists

```ts
const client = new Client({ intents: ["Guilds"] });

client.on("ready", () => {
  console.log(`Logged in as ${client.user?.tag}`);
});

client.login(process.env.TOKEN);
```
